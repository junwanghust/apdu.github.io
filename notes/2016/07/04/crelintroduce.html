CREL Application

2016-07-04

PBOC规范要求所有非接触终端(MSD 和 qPBOC)应使用PPSE(2PAY.SYS.DDF01)目录选择方法。

对于传统的银行IC卡来说只需要内置一个PPSE就可以了，在移动支付时代，一张IC卡上可能会有多家银行的PBOC应用，这就出现了一个问题， 传统IC卡PPSE数据是个人化时写入的，并且不可改写，用户无法选择其中某家银行的PBOC应用来完成支付，而且各家银行的PPSE数据可能不一样，我们需要一个更加灵活的PPSE。

CREL的引入就可以解决这个问题。

定义

一个或多个应用的安装过程中都会涉及到CREL，当应用的注册表有更改后，OPEN都会通知CREL程序。

添加到CREL列表中

INSTALL [for install], INSTALL [for install and make selectable], INSTALL [for registry update]指令，或者使用CRS API。

从CREL列表中删除

INSTALL [for registry update]指令，或者使用CRS API。

简单来说，我们用CREL实例化(INSTALL)一个PPSE之后，我们可以通过INSTALL指令或者CRS来控制某些应用加入到CREL列表(PPSE)中， 对于PPSE FCI中的其它值可以通过INSTALL [for install], INSTALL [for install and make selectable]来传入，这样既可以满足用户选择某家银行的PBOC来支付，又能兼容POS机。

Note：CRS即Contactless Registry Service负责向用户提供检索所有应用组和独立应用的列表，并激活，解除激活或者修改优先级别或者在非接触式界面修改临时的优先级(复位后优先级恢复)。
