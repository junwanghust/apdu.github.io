C#笔记：标识符、基本数据类型与操作符

2019-05-17

1 标识符

1.1 标识符

标识符是程序中对各个元素进行标识的名称，这些元素可以包括命名空间、类、方法和变量。

C#中标识符需要遵循以下规则:

  • 只能使用字母(区分大小写)、数字和下划线
  • 标识符只能以字母或者下划线开头

2 变量

变量是容纳值的存储位置，可以将变量想象成容纳信息的容器，使用该变量就可以获取其对应的值。

2.1 命名变量

通过恰当的命名规范可以来避免混淆、减少bug等，下面是一些常规建议:

  • 不要以下划线开头，虽然在C#中合法，但限制了和其它语言(例如Visual Basic)的互操作性
  • 不要创建仅大小写不同的标识符，例如MyVar与myVar在使用时容易混淆，而且Visual Baisc不区分大小写，重用性也会受限
  • 名称以小写字母开头
  • 在包含多个单词的标识符中，从第二个单词起，每个单词首字母大写(也就是驼峰命名法)，例如footballTeam

2.2 声明变量

声明变量时需要指定变量的类型，例如:

int age;

变量声明时其值为随机值，除非变量被明确赋值，在C/C++未赋值就使用变量可能引起bug，但在C#中不允许使用未赋值的变量，变量只有在赋值后才可使用。

int age;
Console.WriteLine(age); //编译时报错

2.3 隐式声明变量

C#编译器可以根据表达式来推断变量类型，使用var关键字来代替类型即可触发此操作，称之为隐式声明变量。

使用var时需要在声明变量时就要确定变量的类型，类型确定下来后不可更改。

var myVar = 1; //myVar为int
var myDouble = 1.0; //myDouble为double

var myTest; //错误
myVar = "str"; //错误

注意:int var = 1;也是合法的，此处的var不再是上下文关键字。

3 基本数据类型

C#内置了许多基本数据类型(Primitive Data Type)，例如整型、浮点、bool、字符串等。

3.1 整型

  • sbyte类型表示有符号 8 位整数的值介于-128 到 127 之间。
  • byte类型表示其值介于 0 和 255 之间的 8 位无符号的整数。
  • short类型表示有符号 16 位整数值介于-32768 和 32767 之间。
  • ushort类型表示无符号的 16 位整数，其值介于 0 和 65535 之间。
  • int类型表示有符号 32 位整数值介于-2147483648 到 2147483647 之间($-2^{31} \sim 2^{31}-1$)。
  • uint类型表示值介于 0 到 4294967295 之间的 32 位无符号的整数。
  • long类型表示有符号 64 位整数的值介于-9223372036854775808 到 9223372036854775807 之间($-2^{63} \sim 2^{63}-1$)。
  • ulong类型表示值介于 0 到 18446744073709551615 之间的 64 位无符号的整数。
  • char类型表示无符号的 16 位整数，其值介于 0 和 65535 之间。

3.2 浮点类型

C#支持两个浮点类型:float和double。 float为32位单精度,double为64位双精度。

float类型取值范围从大约$1.5 \times 10^{-45}$到$3.4 \times 10^{38}$且精度为7位数字。

double类型取值范围从大约$5.0 \times 10^{-324}$到$1.7 \times 10^{308}$且精度为15到16位。

3.3 十进制类型

decimal类型是适用于财务和货币计算的128位数据类型。 decimal类型可以表示值介于$1.0 \times 10^{-28}$到大约$7.9 \times {10^28}$具有28-29个有效数字。

3.4 Bool 类型

bool类型取值为true或false。

3.5 字符串类型

表示字符串。

int num=10;
long numl=10L;
float numf=10F;
double numd=10.1;
decimal numdec=10M;
char numc='m';
string str = "mmnn";

3.6 NaN与Infinity

C#中任何数除以0所得的结果是无穷大，无穷大不在int、long、decimal范围内，故5/0会报错。 但是double和float实际上有一个可以表示无穷大的特殊值(Infinity)，即5.0/0为Infinity。

0除以任何数其结果为0(被除数为0除外)，如果0.0/0则结果为NaN(Not a Number),代表无效的浮点运算。

NaN和Infinity可以在表达式中使用，10+NaN结果为NaN，10+Infinity结果为Infinity，0*Infinity结果为NaN，NaN*0结果为NaN。

Console.WriteLine(0 / 0);   //编译出错
Console.WriteLine(0.0/0);   //NaN
Console.WriteLine(0.0 / 0 + 10); //NaN
Console.WriteLine(0.0 / 0 * 0);  //NaN

Console.WriteLine(5.0/0);   //Infinity
Console.WriteLine(5.0 / 0 + 10); //Infinity
Console.WriteLine(5.0 / 0 * 0);  //NaN

4 操作符

接受一个操作数的运算符称为一元操作符，例如递增操作符(++)。 接受两个操作数的运算符称为二元操作符，例如算术运算符（*、/、+、- ）。 条件运算符?:接受三个操作数，是 C#中唯一的三元运算符。

4.1 算术操作符

C#支持 +、-、*、/运算，分别代表加、减、乘、除，除此之外还支持%运算符，即取模(余数)运算。

不是所有操作数都适用于任何数据类型，不同数据类型支持的操作符可能不同。

  • char、int、long、float、double、decimal等整型都支持任何算术操作符
  • string仅支持+算术操作符
  • bool类型不支持任何算数操作符
char b = '9';
char c = '2';
Console.WriteLine(b + c);   //107
Console.WriteLine(b % c);   //7
Console.WriteLine(9 % 4);   //1
Console.WriteLine(9 / 4);   //2
Console.WriteLine(9.0 / 4); //2.25
Console.WriteLine(6.2 % 4); //2.2
Console.WriteLine(6.2 + 4); //10.2

注意：C/C++中不允许对浮点类型进行取模运算，但C#允许。

C#中乘法类的操作符(* / %)优先级高于加法类操作符(+ -)，但可用圆括号来覆盖优先级。

4.2 赋值操作符

C#语言中的等号(=)称为赋值操作符，所有操作符都依据他们的操作数返回一个值，赋值操作符也不例外。

赋值操作符取两个操作数：右侧操作数被求值，结果保存在左侧操作数中，赋值操作符的返回值就是赋给左操作数的值。

左操作数 = 右操作数，赋值的同时返回右操作数的值。

赋值操作符的使用如下：

int var1;
int var2;
var1 = var2 = 1;          //将var2 = 1的返回值赋给var1
Console.WriteLine(var2 = 2); //打印返回值2

4.3 递增和递减操作符

给某个变量加1或减1是非常常见的操作，C#专门为这个操作设计了一个操作符:++与–。

++与–是一元操作符，即只有一个操作数。这两个操作符既可以放在变量的前面，也可以放在变量的后面。

int a = 1;
Console.WriteLine(a++);  //1
Console.WriteLine(a--);  //2
Console.WriteLine(++a);  //2
Console.WriteLine(--a);  //1

a++与++a的不同之处在于a++先返回a再加1，++a先加1再返回a，a–与–a同理。

4.4 关系操作符

关系操作符是二元操作符，判断一个值与另一个值的关系，其结果为bool类型：true或false。

操作符 含义 示例 结果
== 等于 100==20 false
!= 不等于 1！=2 true
< 小于 5<6 true
> 大于 6>5 true
>= 小于或等于 5<=5 true
<= 大于或等于 6>=7 false

4.5 逻辑操作符

C#有3个逻辑操作符：&&、||、！（逻辑与，逻辑或，逻辑非）。

逻辑非的作用是将布尔值取反，例如a=false，则!a为true。

逻辑与、逻辑或可以将两个或多个布尔表达式合并成一个布尔值。

只有作为操作数的布尔表达式的值都为true时，&&操作符的结果才为true；操作数其中一个为true时||操作符的结果为true。

&&与||操作符都支持短路求值。有时没有必要为两个操作数都求值，例如当&&的左操作数为false时，不管右操作数的值是什么，整个表达式结果肯定为false。

4.6 复合赋值操作符

复合赋值操作符是在赋值操作符前加一个算术操作符，例如+=、-=、*=、/=、%=。

a += b; //等价a = a+b;
a -= b; //等价a = a-b;
a /= b; //等价a = a/b;
a *= b; //等价a = a*b;
a %= b; //等价a = a%b;

4.7 结合性与优先级

当表达式中出现两个或两个以上具有相同优先级的运算符时，将根据结合性计算它们。 左结合操作符按从左到右的顺序计算，例如：x * y / z 将计算为 (x * y) / z。 右结合操作算符按从右到左的顺序计算，例如：赋值运算符。

三元运算符?:是右结合运算符。 大多数的二元运算符是左结合运算符。

操作符 描述 结合性 优先级
（） 覆盖优先级 左 1
++ 后缀递增 左 1
-- 后缀递减 左 1
! 逻辑取反 左 2
++ 前缀递曾 左 2
-- 前缀递减 左 2
* 乘 左 3
/ 除 左 3
% 取余 左 3
+ 加 左 4
- 减 左 4
< 小于 左 5
<= 小于等于 左 5
> 大于 左 5
>= 大于等于 左 5
== 等于 左 6
!= 不等于 左 6
&& 逻辑与 左 7
|| 逻辑或 左 8
= 赋值 右 9

注：优先级1最高,9最低。
